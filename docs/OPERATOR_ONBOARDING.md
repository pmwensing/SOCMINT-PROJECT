# Operator Onboarding

## Before first use

1. Confirm legal authority and investigation scope.
2. Configure strong secrets and encrypted backups.
3. Run `make ci` and production smoke checks.
4. Review `/api/v1/production-release/summary`.
5. Review `/api/v1/admin/security/checklist`.
6. Review `/api/v1/admin/gates/enforcement/summary`.
7. Configure team and case access controls.

## During a case

1. Create or select the correct case.
2. Record purpose, scope, and authorization.
3. Add seeds only within scope.
4. Run only approved connectors.
5. Review observations before promoting assertions.
6. Use evidence capture and custody notes.
7. Run export preflight before sharing.

## Before export

1. Confirm redaction coverage.
2. Confirm source and evidence traceability.
3. Confirm no rejected claims are included.
4. Save manifest and custody summary.
5. Preserve final bundle hash.
