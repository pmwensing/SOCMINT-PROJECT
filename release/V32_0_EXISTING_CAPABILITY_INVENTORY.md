# v32.0 — Existing Capability Inventory

v32 coordinates existing publication and distribution capabilities. It does not replace them.

## Reused v31 publication capabilities

- publication candidates
- draft revision assembly
- editorial validation
- human release approval
- immutable published revisions
- publication supersession history

The immutable v31 published revision remains the only valid source for a v32 dissemination package.

## Reused v22 delivery capabilities

- release workspace
- release authorization
- release preview
- secure distribution
- delivery receipts
- release history

## Implemented v32 coordination layers

- v32.1 audience and recipient contracts
- v32.2 deterministic package assembly
- v32.3 human authorization and policy decisions
- v32.4 append-only delivery attempts and receipts
- v32.5 recipient feedback and correction intake separated from source intelligence

## Remaining gaps

- recall, retention, and lifecycle history
- unified product review and browser closure gate

## Non-goals

- no second publication system
- no replacement delivery engine
- no automatic recipient authorization
- no automatic external transmission
- no feedback-driven rewrite of source intelligence
- no mutation or deletion of publication or delivery history
- no schema migration without a proven storage gap
